## 4. 矿业系统

### 基础参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 基础每季产金量 | 2 单位 | 每座矿山 |
| 基础每季产银量 | 5 单位 | 每座矿山 |
| 黄金售价 | 50 / 单位 | 受通胀乘数影响 |
| 白银售价 | 10 / 单位 | 受通胀乘数影响 |
| 矿山最大等级 | 5 | — |
| 升级基础费用 | 300 × 等级 | 隐含在升级逻辑中 |
| 每级产能加成 | +20% | 乘法叠加 |
| 事故基础概率 | 5% / 季 | 受治安和投入影响 |

### 产金公式

```
单矿产金 = (base_gold_output + mine_output_modifier) × levelMultiplier × (1 + positionBonus) + workerBonus

其中：
  base_gold_output = 2
  mine_output_modifier = 来自科技/事件的 GetModifierValue("mine_output")
  levelMultiplier = 1.0 + (mine.level - 1) × 0.20
  positionBonus = 矿业总监岗位加成 (满配 1.0 / 半配 0.5 / 差配 -0.1 / 空缺 -0.30)
  workerBonus = floor(hired_workers / 10)
```

> 产金受地区黄金储量 (`gold_reserve`) 限制，每季开采量不超过剩余储量。开采后储量减少。

### 产银公式

```
单矿产银 = floor(base_silver_output × (1 + (mine.level - 1) × 0.20))
```

> 白银不消耗储量（不占 `gold_reserve`），每季自动全量售出。

### 黄金出售

| 模式 | 行为 |
|------|------|
| 手动（默认） | 玩家在产业页手动出售，保留全部库存 |
| 自动出售 | 开启后，每季结算时保留库存的 **10%**（四舍五入），出售剩余黄金 |

自动出售价格 = `gold_price × inflation × (1 + military_industry_profit_modifier × 0.5) × (1 + gold_price_bonus) × (1 + gold_price_mod)`

> 注意：白银每季出售时会消耗 `silver_reserve`（储量），与文档原描述"不占储量"不同。

## 7. 经济结算系统

每季结算按以下顺序执行（`Economy.Settle()`）：

### 收入项

| 来源 | 计算方式 |
|------|----------|
| 黄金出售收入 | 仅在 `gold_auto_sell = true` 时：保留 10%（四舍五入），出售剩余。价格 = `gold_price × inflation × 军需修正 × 科技修正 × 事件修正` |
| 白银出售收入 | 全量出售（消耗 silver_reserve）：`silver × silver_price × inflation × (1 + silver_price_mod)` |
| 煤炭出售收入 | 工业区每季采煤并全量出售：`coal × coal_price(5) × inflation × (1 + coal_price_mod)` |
| 地区被动收入 | 见下方"地区控制度被动收入" |
| 金融被动收入 | 科技"金融网络"解锁后的被动收入 |
| 贸易被动收入 | 科技"贸易路线"解锁后的被动收入 |
| 影子收入 | 事件修正器 `shadow_income` 提供 |

> **黄金自动出售保留比例**：保留当前黄金库存的 **10%**（`math.floor(gold × 0.1 + 0.5)` 四舍五入），而非固定保留 10 单位。

### 煤炭系统

| 参数 | 值 |
|------|-----|
| 基础每季采煤量 | 8 单位（工业区） |
| 煤炭单价 | 5 / 单位 |
| 采煤公式 | `floor(base_coal_output × (1 + (development - 1) × 0.15))` |
| 储量消耗 | 每季采出量从 `coal_reserve` 中扣除 |

> 煤炭仅在工业区（`type = "industrial"`）且有 `coal_reserve > 0` 时采集，全量出售。

### 地区控制度被动收入

| 地区类型 | 触发条件 | 收入公式 |
|----------|----------|----------|
| 矿区 (mine) | 控制度 ≥ 50% | `floor((control - 50) / 10) × 20 × inflation` |
| 工业城 (industrial) | 控制度 ≥ 40% | `floor(control × 1.5 × inflation × max(0.4, 1 + civilian_demand))` |
| 首都 (capital) | 控制度 ≥ 30% | 减税效果：每超出 10% → 税率 -1% |

### AI 存在度负面效果

| 条件 | 效果 |
|------|------|
| 任意 AI 存在度 ≥ 50% | 每超出 10% → 额外支出 +15 × inflation |

### 支出项

| 类别 | 计算方式 |
|------|----------|
| 工人工资 | `hired × wage × laborCostFactor × hireCostMul` |
| 军事开支 | `guards × guard_wage × inflation` |
| 补给开支 | `guards × supply_per_guard(3) × supply_cost(2) × inflation × supplyDiscount × (1 + transportRisk)` |
| 税收 | `cash × tax_rate`（和平 5% / 战时 12% + 事件修正 + 首都减税 + 合法性/腐败修正） |
| AI 存在惩罚 | 见上方"AI 存在度负面效果" |
| 运输风险 | `exposedIncome × min(0.30, transportRisk × 0.25)` |

### 收入修正器

全部总收入受事件修正器 `income_mod` 影响（范围 -75% ~ +100%）：
```
adjustment = floor(grossIncome × clamp(income_mod, -0.75, 1.00))
```

### 净收入

```
net = total_income - total_expense
cash = cash + total_income（已在出售环节加入）
cash = cash - total_expense
```

### 破产处理

当 `cash < 0` 时：
1. 紧急变卖黄金：`sellGold = min(gold, ceil(|cash| / gold_price))`
2. 仍为负则归零：`cash = 0`

| 基础因子 | 1.0 | 开局无通胀 |
| 和平漂移 | +0.006 / 季 | +0.6%/季，年化约 +2.4%（接近历史） |
| 战时漂移 | +0.12 / 季 | +12%/季（恶性通胀：16季 ≈ 1.12^16 ≈ 6×，加事件可达 10-15×） |
| 通缩漂移下限 | -0.04 / 季 | 大萧条/战后通缩可达 -4%/季（年化 -15%） |
| 上限 | 20.0 | 最多 20 倍物价（南斯拉夫二战占领区历史参考） |
| 下限 | 0.45 | 大萧条期间可压至基准 0.45×（历史通缩 -20~-30%） |

### 资产价格修正

| 参数 | 值 | 说明 |
|------|-----|------|
| 资产暴跌下限 | -60% | 大萧条期间资产价格可跌 60%（接近历史股市跌幅） |
| 资产泡沫上限 | +100% | 战时资产泡沫可达 +100%（军需品价格翻倍） |

### 通胀影响

- 黄金/白银/煤炭售价 × inflation
- 工人工资 × inflation × laborCostFactor
- 护卫工资 × inflation
- 补给费用 × inflation
- 地区被动收入 × inflation

> 通胀**可逆**：和平时温和上涨，战时恶性通胀，大萧条/战后可发生通缩。通胀漂移受事件修正器 `inflation_drift` 影响，AI 外资也可操纵通胀。

## 9. 贷款系统

### 基础参数

| 参数 | 值 |
|------|-----|
| 同时最多持有 | 3 笔 |
| 违约惩罚 | 本金膨胀 +15% |
| 最多展期次数 | 1 次 |
| 坏账核销士气惩罚 | -10 |
| 最大杠杆率 | 80%（超过禁止再贷款） |

### 抵押估值

贷款额度基于玩家资产估值，而非固定金额：

| 资产类型 | 抵押率 |
|----------|--------|
| 实体资产（现金/黄金/矿山等） | 80% |
| 股票市值 | 25% |

`抵押价值 = 实体资产 × 0.80 + 股票市值 × 0.25`

### 可选贷款档

| 档位 | 额度比例 | 保底金额 | 基础季利率 | 期限（季） |
|------|---------|---------|-----------|-----------|
| 小额短贷 | 抵押价值 × 15% | 300 | 4% | 4 |
| 中额贷款 | 抵押价值 × 35% | 800 | 5% | 6 |
| 大额长贷 | 抵押价值 × 60% | 2,000 | 6% | 8 |

### 杠杆利率

实际利率随负债水平动态上浮：

```
实际利率 = base_interest × (1 + leverage × 1.5)
leverage = 总负债 / 抵押价值
```

示例：杠杆率 0.5 → 利率 ×1.75；杠杆率 1.0 → 利率 ×2.5

### 结算逻辑（每季）

1. **付息**：`interest = ceil(principal × 实际利率)`
   - 付得起 → 扣现金，累计 `total_paid`
   - 付不起 → 启动**强制抵押清算**：
     1. 第一步：强制变卖黄金偿付利息
     2. 第二步：黄金不够则降级矿山换现金（退还升级费的 50%）
     3. 每次清算额外扣士气 -5
     4. 清算后凑够 → 正常扣款
     5. 清算后仍不够 → 真正违约，本金 × 1.15 膨胀
2. **到期还本**：
   - 付得起 → 清偿，贷款消失
   - 付不起 + 未展期过 → 部分清偿 + 延长 4 季 + 本金 × 1.15
   - 付不起 + 已展期 → 强制清算偿还，仍不够则坏账核销（士气 -10）

### 破产条件

| 条件 | 阈值 |
|------|------|
| 连续违约（强制清算后仍无法偿付） | 4 季 → 触发破产 |
| 净资产连续为负 | 4 季 → 触发破产 |
| 预警 | 连续违约 2 季开始警告 |

## 10. 股市系统（GBM）

### 核心模型：几何布朗运动

```
P(t+1) = P(t) × exp((mu - sigma²/2) × dt + sigma × eps × sqrt(dt))

其中 eps ~ N(0,1) 由 Box-Muller 变换生成
dt = 1（每季度推进一次）
```

### 三层修正

| 层级 | 来源 | 说明 |
|------|------|------|
| L1 基本面 | 每支股票自带 mu / sigma | 长期趋势 |
| L2 事件修正 | `event_mu_mods` delta_mu | 短期冲击，有持续季度数 |
| L3 战时放大 | sigma × 1.8 | 战争章节全局生效 |

### AI 每季行为

1. **资产增长**：`cash += floor(cash × growth_rate)`（受情报渗透 debuff 影响，受 cash_cap 限制）
2. **势力增长**：和平时每季 **+2**，战时每季 **+3**；现金超过扩张阈值时额外 +1，同时向目标地区扩张存在度 +2
3. **协议保护**：签约期内 AI 不敌对，态度不低于 10
4. **外资战时撤资**：战时有概率撤走 15% 现金，势力 -3
5. **AI 主动花费**：见下方"AI 花费行为"表
6. **态度变动**：负向触发器 + 正向触发器（见下方）

| 经济制裁 | 600 | 玩家收入 -10%（3季） | 仅外资，attitude < -40 | 15% |

| 矿价波动 | 700 | 金银价格 -15%（3季） | 仅外资，attitude < -35 | 15% |

## 15. 战斗系统

### 玩家战力计算

```
PlayerPower = guards × guard_base_power(1.0)
            × max(0.3, morale × morale_multiplier(0.01))
            × (1 + (equipment - 1) × equipment_bonus(0.15))
            × (1 + chiefBonus)
```

示例：5 护卫、70 士气、1 级装备、无军务主管 → `5 × 1.0 × 0.7 × 1.0 × 1.0 = 3.5`

### AI 战力

```
FactionPower = faction.power × 1.0
```

### 战斗判定

- 双方各 ×(0.8 + random × 0.4) 随机因子（±20% 波动）
- 比较 pRoll vs aRoll，大者胜

### 战斗结果

| 结果 | 玩家胜 | 玩家败 |
|------|--------|--------|
| 现金 | +AI 现金 × 25% | -玩家现金 × 10% |
| 护卫 | 无损失 | -30% 护卫 |
| 士气 | +10 | -18 |
| AI 势力 | power -8, attitude -10 | power +5 |
| 地图影响 | 冲突地区: 控制度+3, AI存在度-8 | 冲突地区: 控制度-5, AI存在度+7, 治安-1 |
| 胜场计数 | `battle_wins_unclaimed +1`（每季结算时计入军事胜利点） | AI 累计 `battle_wins_unclaimed +1` |

> 注意：军事胜利点不在战斗时直接增加，而是通过每季回合结算阶段 3 中 `min(unclaimed, 3)` 计入。

### AI 主动进攻条件

| 条件 | 阈值 |
|------|------|
| AI 态度 | ≤ -20 (`ai_attack_threshold = -20`) |
| AI 势力值 | ≥ 40 (`ai_attack_power_req = 40`) |
| 每季概率 | 35% 基础 × (1 + aggression)，上限 85% |
| 协议保护 | 保护期内不进攻 |

> AI 进攻概率公式：`chance = ai_attack_chance(0.35) × (1 + aiConfig.aggression)`，其中地方家族 aggression=0.3、外资 aggression=0.1。实际概率 `math.min(0.85, chance)`。

## 18. 胜利条件判定

### 三种胜利方式

| 方式 | 条件 | 说明 |
|------|------|------|
| **经济绝对胜利** | 经济胜利点 ≥ 1600 + 快照验证 | 1930 年后才开始结算 |
| **军事绝对胜利** | 军事胜利点 ≥ 2000 + 快照验证 | 1925 年后才开始结算 |
| **相对领先胜利** | 领先 AI 达到一定幅度 | 1945 年后可宣布 |

### 经济胜利点 — 每季结算公式

```lua
ecoDelta = floor(cash / 2000) + floor(gold × 0.5) + floor(totalControl / 15) + floor(totalInfluence / 50)
-- 战时：ecoDelta = floor(ecoDelta × 0.6)
-- 影响力 ≥ 300 时：ecoDelta += 5
```

| 来源 | 公式 | 示例 |
|------|------|------|
| 现金贡献 | `floor(cash / 2000)` | 10000 现金 → +5 点 |
| 黄金贡献 | `floor(gold × 0.5)` | 20 黄金 → +10 点 |
| 地区控制 | `floor(totalControl / 15)` | 总控制 105 → +7 点 |
| 影响力 | `floor(totalInfluence / 50)` | 总影响力 100 → +2 点 |
| 战时修正 | × 0.6（战争拖慢经济胜利） | — |
| 影响力阈值加成 | 总影响力 ≥ 300 时 +5/季 | — |

> **章节门控**：经济线要到 **1930 年**之后才开始结算胜利点。

### 经济胜利快照验证

达到 1600 点阈值的瞬间，必须同时满足以下条件，否则视为无效：

| 条件 | 最低值 |
|------|--------|
| 现金 | ≥ 10,000 |
| 黄金 | ≥ 20 |
| 总控制度 | ≥ 80 |

### 军事胜利点 — 每季结算公式

```lua
milDelta = floor(guards × 0.3) + floor(morale / 25) + floor(totalControl / 12) + min(battleWinsUnclaimed, 3)
-- 战时：milDelta = floor(milDelta × 1.25)
-- 影响力 ≥ 300 时：milDelta += 5
```

| 来源 | 公式 | 示例 |
|------|------|------|
| 护卫贡献 | `floor(guards × 0.3)` | 25 护卫 → +7 点 |
| 士气贡献 | `floor(morale / 25)` | 75 士气 → +3 点 |
| 地区控制 | `floor(totalControl / 12)` | 总控制 105 → +8 点 |
| 近期战斗胜利 | `min(unclaimed, 3)` | 每季最多计入 3 场 |
| 战时修正 | × 1.25（战争加速军事胜利） | — |
| 影响力阈值加成 | 总影响力 ≥ 300 时 +5/季 | — |

> **章节门控**：军事线要到 **1925 年**之后才开始结算胜利点。

## 19. 回合结算完整流程

`TurnEngine.EndTurn(state)` 共 15+ 个阶段：

| 阶段 | 名称 | 关键操作 |
|------|------|----------|
| 0 | 通胀推进 | inflation_factor += drift（和平 +0.006 / 战时 +0.12）+ 事件修正，clamp [0.45, 20.0] |
| 1 | 经济结算 | Economy.Settle()：产矿（金+银+煤） → 出售 → 地区被动收入 → 工资 → 税收 → AI惩罚 → 破产检查 |
| 1.5 | 股市更新 | StockEngine.UpdateAll()：GBM 推进 + 事件修正消耗 |
| 1.6 | 贷款结算 | 动态利率计算 → 付息 → 强制抵押清算 → 到期还本/展期 → 破产检测 |
| 1.7 | 科技推进 | Tech.Tick()：进度 +1（科技顾问可能额外 +1） |
| 2 | 事件检查 | 冷却推进 → 触发判定 → 入队 |
| 3 | **胜利点结算** | 经济/军事胜利点累加 + 章节门控 + war_mod + AI 同步累积（详见第 18 节） |
| 4 | 修正器推进 | 持续效果倒计时，到期移除 |
| 4.5 | **影响力衰减** | 所有地区 influence -1/季（除非本季执行了文化行动） |
| 4.5+ | **监管检查** | regulation_pressure ≥ 50 时有概率罚没现金 3% |
| 5 | 士气衰减 | 护卫士气 -2（军务主管减缓） |
| 6 | AI 更新 | 资产增长(cash_cap) → 势力增长(+2/+3) → AI花费行为 → 态度触发器 → 外资撤退 |
| 6.2 | 外资控制度修正 | 根据 foreign_capital 地区存在度计算 foreign_control 修正器 |
| 6.5 | AI 战斗 | Combat.ResolveAIActions()：满足条件时 AI 进攻 |
| 6.7 | **大国博弈** | GrandPowers.Tick()：历史漂移 → 继承处理 → 征服执行 → 抵抗增长 → 本地AI联动 |
| 6.8 | **分支事件** | BranchEvents.CheckBranchEvents()：检查历史分支节点 → 入队 |
| 7 | 工人士气 | 工资检查 → 士气变动 → 罢工警告 |
| 8 | 家族培养 | 培养进度 +1 → 完成则新成员加入 |
| 9 | 推进季度 | year/quarter 推进 → AP 重置 → 日志记录 |

## 20. 设计问题追踪：地图控制与胜利的关联

> **此为体验游戏过程中发现的设计缺陷及修复记录。**

### 原始问题

地图系统提供了丰富的战略数据（控制度、AI 存在、影响力），但早期版本中这些数据与胜利关联微弱。

### 已修复项目 ✅

以下问题已在当前代码版本中修复：

| 原始问题 | 修复方案 | 涉及系统 |
|----------|----------|----------|
| 经济胜利与地区控制无关 | 经济胜利点新增 `floor(totalControl / 15)` 和 `floor(totalInfluence / 50)` | §18 胜利条件 |
| 影响力无实际用途 | ① 影响力 ≥ 300 时经济/军事双线各 +5/季；② 影响力自然衰减 -1/季（需持续投入）；③ 影响力 5 级被动效果（见 balance.lua INFLUENCE.thresholds） | §18 胜利条件、回合结算 |
| 控制度不影响税率 | 首都控制度 ≥ 30% 时减税（每超 10% → 税率 -1%） | §7 经济结算 |
| 控制度不提供经济收入 | 矿区 ≥50%、工业城 ≥40% 时提供被动收入 | §7 经济结算 |
| AI 存在度无实际威胁 | AI 存在度 ≥50% 时每超 10% → 额外支出 +15 × inflation | §7 经济结算 |
| AI 只堆积现金不花费 | 新增 5 种 AI 花费行为（雇佣兵/压制/制裁/通胀操纵/矿价波动） | §14 AI 势力 |
| AI 态度单调下降 | 新增 4 种正向态度触发器 + 自然回暖 | §14 AI 势力 |

### 仍存在的限制

| 方面 | 当前状态 | 潜在改进方向 |
|------|----------|-------------|
| 矿山产出 | 控制度不直接改变产金量 | 可考虑控制度低于阈值时降低效率 |
| 工人/护卫招募上限 | 不受人口或控制度影响 | 可考虑与地区人口挂钩 |
| 事件触发 | 控制度不改变事件概率 | 可考虑低控制度增加负面事件概率 |

## 21. 影响力系统

影响力 (Influence) 是地区级别的战略资源，需要持续投入维护，并在高值时提供被动效果和胜利点加成。

### 基础参数

| 来源：政治渗透 | +2 / 次 | 行动增益 |

## 22. 幸运事件系统（广告激励）

玩家观看广告后可触发幸运事件，获得现金奖励。奖励金额随通胀浮动，概率逐次递减。

### 奖励档位

| 档位 | 基础金额 | 初始权重 | 说明 |
|------|---------|---------|------|
| 小额意外之财 | 666 | 50 | 最常见 |
| 商队赠礼 | 888 | 30 | — |
| 矿脉意外发现 | 1,200 | 12 | — |
| 贵族赞助 | 1,600 | 5 | — |
| 中了头彩！ | 2,000 | 3 | 最稀有 |

> 实际奖励 = `base × inflation_factor`

### 概率递减机制

| 参数 | 值 | 说明 |
|------|-----|------|
| 衰减因子 | 0.7 | 每次观看后各档权重 × 0.7 |
| 最低衰减 | 15% | 权重最低降至原始的 15% |
| 每季上限 | 3 次 | 同一季度最多观看 3 次广告 |
