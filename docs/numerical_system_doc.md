## 4. 矿业系统

### 基础参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 基础每季产金量 | 2 单位 | 每座矿山 |
| 基础每季产银量 | 5 单位 | 每座矿山 |
| 黄金售价 | 50 / 单位 | 受通胀乘数影响 |
| 白银售价 | 10 / 单位 | 受通胀乘数影响 |
| 矿山最大等级 | 5 | — |
| 升级基础费用 | 300 × 等级 | 隐含在升级逻辑中 |
| 每级产能加成 | +20% | 乘法叠加 |
| 事故基础概率 | 5% / 季 | 受治安和投入影响 |

### 产金公式

```
单矿产金 = (base_gold_output + mine_output_modifier) × levelMultiplier × (1 + positionBonus) + workerBonus

其中：
  base_gold_output = 2
  mine_output_modifier = 来自科技/事件的 GetModifierValue("mine_output")
  levelMultiplier = 1.0 + (mine.level - 1) × 0.20
  positionBonus = 矿业总监岗位加成 (满配 1.0 / 半配 0.5 / 差配 -0.1 / 空缺 -0.30)
  workerBonus = floor(hired_workers / 10)
```

> 产金受地区黄金储量 (`gold_reserve`) 限制，每季开采量不超过剩余储量。开采后储量减少。

### 产银公式

```
单矿产银 = floor(base_silver_output × (1 + (mine.level - 1) × 0.20))
```

> 白银不消耗储量（不占 `gold_reserve`），每季自动全量售出。

### 黄金出售

| 模式 | 行为 |
|------|------|
| 手动（默认） | 玩家在产业页手动出售，保留全部库存 |
| 自动出售 | 开启后，每季结算时自动出售超过 10 单位的黄金 |

自动出售价格 = `gold_price × inflation_factor × (1 + military_industry_profit_modifier × 0.5)`

## 7. 经济结算系统

每季结算按以下顺序执行（`Economy.Settle()`）：

### 收入项

| 来源 | 计算方式 |
|------|----------|
| 黄金出售收入 | 仅在 `gold_auto_sell = true` 时：`(gold - 10) × gold_price × inflation × 军需修正` |
| 白银出售收入 | 全量出售：`silver × silver_price × inflation` |

### 支出项

| 类别 | 计算方式 |
|------|----------|
| 工人工资 | `hired × wage × inflation` |
| 军事开支 | `guards × guard_wage × inflation` |
| 补给开支 | `guards × supply_per_guard(3) × supply_cost(2) × inflation` |
| 税收 | `cash × tax_rate`（和平 5% / 战时 12% + 事件修正） |

### 净收入

```
net = total_income - total_expense
cash = cash + total_income（已在出售环节加入）
cash = cash - total_expense
```

### 破产处理

当 `cash < 0` 时：
1. 紧急变卖黄金：`sellGold = min(gold, ceil(|cash| / gold_price))`
2. 仍为负则归零：`cash = 0`

| 参数 | 值 | 说明 |
|------|-----|------|
| 基础因子 | 1.0 | 开局无通胀 |

| 战时漂移 | +0.025 / 季 | +2.5%，4 年战争通胀约 +40% |
| 上限 | 4.0 | 最多 4 倍物价 |

- 黄金/白银售价 × inflation
- 工人工资 × inflation
- 护卫工资 × inflation
- 补给费用 × inflation

> 通胀是**不可逆**的，只增不减，战时急剧攀升。

| 小额 | 500 | 6% | 4 |

### 结算逻辑（每季）

1. **付息**：`interest = ceil(principal × interest_rate)`
   - 付得起 → 扣现金，累计 `total_paid`
   - 付不起 → 违约，本金 × 1.10 膨胀
2. **到期还本**：
   - 付得起 → 清偿，贷款消失
   - 付不起 → 部分清偿（用光现金），剩余展期 4 季，本金再 × 1.10

## 10. 股市系统（GBM）

### 核心模型：几何布朗运动

```
P(t+1) = P(t) × exp((mu - sigma²/2) × dt + sigma × eps × sqrt(dt))

其中 eps ~ N(0,1) 由 Box-Muller 变换生成
dt = 1（每季度推进一次）
```

### 三层修正

| 层级 | 来源 | 说明 |
|------|------|------|
| L1 基本面 | 每支股票自带 mu / sigma | 长期趋势 |
| L2 事件修正 | `event_mu_mods` delta_mu | 短期冲击，有持续季度数 |
| L3 战时放大 | sigma × 1.8 | 战争章节全局生效 |

## 15. 战斗系统

### 玩家战力计算

```
PlayerPower = guards × guard_base_power(1.0)
            × max(0.3, morale × morale_multiplier(0.01))
            × (1 + (equipment - 1) × equipment_bonus(0.15))
            × (1 + chiefBonus)
```

示例：5 护卫、70 士气、1 级装备、无军务主管 → `5 × 1.0 × 0.7 × 1.0 × 1.0 = 3.5`

### AI 战力

```
FactionPower = faction.power × 1.0
```

### 战斗判定

- 比较 pRoll vs aRoll，大者胜

## 18. 胜利条件判定

### 胜利阈值

**经济胜利或军事胜利任一达到 100 点即获胜。** 上限锁定为 120（threshold + 20）。

### 经济胜利点 — 每季结算公式

```lua
ecoDelta = floor(cash / 500) + gold × 2
```

| 来源 | 公式 | 示例 |
|------|------|------|
| 现金贡献 | `floor(cash / 500)` | 5000 现金 → +10 点 |
| 黄金贡献 | `gold × 2` | 20 黄金 → +40 点 |
| **合计** | ecoDelta = 现金项 + 黄金项 | — |

> **注意**：经济胜利只看现金和黄金，**不包含**股票持仓市值、地区控制度、科技数量或任何其他资产。

### 军事胜利点 — 每季结算公式

```lua
milDelta = floor(guards × 0.5) + floor(morale / 20) + SUM(floor(region.control / 10))
```

| 来源 | 公式 | 示例 |
|------|------|------|
| 护卫贡献 | `floor(guards × 0.5)` | 20 护卫 → +10 点 |
| 士气贡献 | `floor(morale / 20)` | 80 士气 → +4 点 |
| 地区控制贡献 | `SUM(floor(region.control / 10))` | 80%+20%+5% → 8+2+0 = +10 点 |
| 战斗胜利奖励 | 每次战斗胜利 +5 | 在 `Combat.ApplyResult()` 中直接加 |

### 胜利点累积速度分析

以典型中期状态为例（现金 3000，黄金 15，护卫 15，士气 70，控制度 80/20/5）：

| 每季 | 经济胜利点 | 军事胜利点 |
|------|-----------|-----------|
| 增量 | 6 + 30 = **36** | 7 + 3 + 10 = **20** |
| 达到 100 需 | ~3 季 | ~5 季 |

> 以上是**累加**模型，胜利点每季都在叠加，不会减少。随着游戏推进，两条路线的增速会越来越快。

### 游戏结束条件

| 条件 | 类型 |
|------|------|
| 经济胜利点 ≥ 100 | 经济胜利 |
| 军事胜利点 ≥ 100 | 军事胜利 |
| 时间超过 2014 Q4 | 超时结束（无胜利） |

## 20. 设计问题：地图控制与胜利脱节

> **此为体验游戏过程中发现的设计缺陷。**

### 问题描述

当前地图系统提供了 3 个地区的控制度、AI 存在、影响力等丰富的战略数据，但这些数据与游戏最终胜利的关联**极其微弱**。

### 具体分析

#### 经济胜利路线 — 完全无关

经济胜利点公式为 `floor(cash / 500) + gold × 2`，**完全不涉及**：
- 地区控制度
- 地区影响力
- AI 在地区的存在度
- 地区基建/治安/人口

玩家即使在工业区和首都的控制度为 0%，只要现金和黄金足够，照样可以达成经济胜利。

#### 军事胜利路线 — 联系微弱

军事胜利点中的地区控制贡献为 `SUM(floor(region.control / 10))`：

| 场景 | 三区控制度 | 每季贡献 | 占比 |
|------|-----------|---------|------|
| 初始状态 | 80/20/5 | +10 | 约 50% |
| 全力渗透后 | 100/80/60 | +23 | 约 55% |
| 完全不管 | 80/20/5 | +10 | — |

虽然控制度确实贡献军事胜利点，但：
1. **增量有限**：从不管到全力渗透，仅增加 +13 点/季
2. **初始控制度已经不低**：矿区 80% 已贡献 +8
3. **投入产出不成正比**：每次渗透消耗 2 AP，总共只增加少量胜利点

#### 控制度缺乏实际战略价值

更关键的是，**地区控制度不影响任何其他游戏系统**：

| 应该影响但目前未影响的系统 | 说明 |
|--------------------------|------|
| 矿山产出 | 控制度高低不改变产金量 |
| 税率 | 控制度不影响税收 |
| 工人招募上限 | 不受人口或控制度影响 |
| 护卫招募上限 | 同上 |
| 贸易机会 | 工业区控制度不解锁新的收入来源 |
| 外交筹码 | 首都控制度不提供外交优势 |
| 事件触发 | 控制度不改变事件概率或选项 |
| 科技研发 | 不受控制度影响 |

#### 影响力 (influence) 字段

科技"印刷宣传"每季给所有地区 +2 影响力，政治渗透也给 +2 影响力，但 **influence 字段在整个游戏中没有被任何系统消费或参与计算**。它仅仅是一个持续增长的数字，没有实际游戏效果。

### 改进建议

为使地图控制系统具有战略意义，建议考虑以下方向：

1. **经济挂钩**：高控制度地区提供额外税收/贸易收入，或解锁新的产业
2. **资源解锁**：工业区控制度 ≥ 50% 时解锁钢铁加工产能，首都控制度 ≥ 30% 时获得政治资源
3. **影响力消费**：影响力可用于外交行动、降低科技成本、或转化为胜利点
4. **控制度直接计入经济胜利**：如每 10% 控制度 = +1 经济胜利点
5. **AI 存在度的实际威胁**：AI 在地区的高存在度应产生负面效果（如税率上升、事故概率增加）
6. **地区事件**：控制度低的地区触发负面事件概率更高
