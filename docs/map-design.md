# 欧洲地图设计文档

> 百年萨拉热窝：黄金家族 — 交互式欧洲地图控件

---

## 1. 整体架构

```
ui_map_widget.lua    ← 地图控件（Widget:Extend，NanoVG 渲染）
ui_world.lua         ← 世界页（集成地图 + 详情面板 + AI势力 + 历史日志）
```

地图控件是一个自定义 Widget，使用 NanoVG 多边形渲染，支持缩放/平移/点击交互。世界页通过 `MapWidget` 公开 API 集成地图并响应区域选择事件。

## 2. 坐标系统

### 2.1 世界坐标

所有区域多边形使用 **归一化坐标 (0~1)**，映射真实经纬度范围：

| 轴 | 范围 | 含义 |
|----|------|------|
| X  | 0 → 1 | 15°W → 45°E（西到东） |
| Y  | 0 → 1 | 71°N → 34°N（北到南，Y=0 在上方） |

### 2.2 坐标变换

```
世界坐标 (wx, wy)  ←→  屏幕像素 (sx, sy)
```

**世界 → 屏幕（_W2S）**：
```
sx = mx + (wx - camX) * zoom * mw + mw * 0.5
sy = my + (wy - camY) * zoom * mh + mh * 0.5
```

**屏幕 → 世界（_S2W）**：
```
wx = (sx - mx - mw * 0.5) / (zoom * mw) + camX
wy = (sy - my - mh * 0.5) / (zoom * mh) + camY
```

其中 `(mx, my, mw, mh)` 是控件绘制区域（扣除内边距），`(camX, camY)` 是相机中心的世界坐标，`zoom` 是缩放倍数。

## 4. 缩放/平移系统

### 4.1 参数常量

| 常量 | 值 | 含义 |
|------|----|------|
| MIN_ZOOM | 1.0 | 最小缩放（完整欧洲） |
| MAX_ZOOM | 10.0 | 最大缩放（波黑细节） |
| ZOOM_STEP | 1.35 | 每次缩放的倍率 |
| DRAG_THRESH | 5 px | 拖拽阈值（区分点击和拖拽） |
| BOSNIA_CX | 0.510 | 波黑聚焦中心 X |
| BOSNIA_CY | 0.730 | 波黑聚焦中心 Y |
| BOSNIA_ZOOM | 7.5 | 波黑聚焦缩放级别 |

### 4.2 相机状态

```lua
self.zoom_  -- 当前缩放倍数 [1.0, 10.0]
self.camX_  -- 相机中心 X（世界坐标）
self.camY_  -- 相机中心 Y（世界坐标）
```

默认值：`zoom=7.5, camX=0.510, camY=0.730`（聚焦波黑）

### 4.3 缩放策略：Zoom-at-Point

缩放时保持鼠标/触点下的世界坐标不变：

```
1. 将屏幕坐标转换为世界坐标 (wx, wy)
2. 应用新的 zoom 值
3. 反推新的 camX/camY 使 (wx, wy) 仍映射到同一屏幕位置：
   camX_new = wx - (wx - camX_old) * zoom_old / zoom_new
   camY_new = wy - (wy - camY_old) * zoom_old / zoom_new
4. 钳制相机不超出边界
```

### 4.4 相机边界钳制

确保视口不超出世界 [0,1] 范围：

```
halfW = 0.5 / zoom
halfH = 0.5 / zoom
camX ∈ [halfW, 1 - halfW]
camY ∈ [halfH, 1 - halfH]
```

## 5. 交互操作

### 5.1 操作方式

| 操作 | 实现方式 | 说明 |
|------|---------|------|
| 滚轮缩放 | `input:GetMouseMoveWheel()` 在 Render 中轮询 | 以光标位置为中心缩放 |
| 拖拽平移 | OnPointerDown/Move/Up | 5px 阈值区分点击，zoom>1 时捕获事件 |
| 点击选区 | OnPointerUp（无拖拽时） | 射线法 Point-in-Polygon 检测 |
| 悬停高亮 | OnPointerMove（非拖拽时） | 只检测游戏区域 |
| [+] 按钮 | 以控件中心放大 1.35x | — |
| [−] 按钮 | 以控件中心缩小 1/1.35x | — |
| [🌍] 按钮 | zoom=1, cam=(0.5, 0.5) | 全欧洲视图 |
| [◎] 按钮 | zoom=7.5, cam=(0.51, 0.73) | 聚焦波黑 |

### 5.2 拖拽与 ScrollView 冲突处理

- `zoom > 1.05` 时，地图捕获指针事件（`StopPropagation`），支持拖拽平移
- `zoom ≈ 1.0` 时，事件向上冒泡，ScrollView 正常滚动

### 5.3 Point-in-Polygon 碰撞检测

使用射线法（Ray Casting），在屏幕坐标空间进行检测：

```
对于点 P，从 P 向右发射水平射线
统计与多边形边的交点数
奇数 = 在内部，偶数 = 在外部
```

多边形顶点存储为世界坐标，碰撞检测时先转为屏幕坐标再判断。

## 6. 渲染层次

由下到上的绘制顺序：

```
1. 深色底板（nvgRoundedRect, 深海色 rgba(30,34,42)）
2. 裁剪区域（nvgScissor）
3. 经纬网格（每 0.1 世界单位一条线）
4. 欧洲背景区域多边形（半透明填充 + 暗色描边）
5. 欧洲背景标签（字号/透明度随缩放自适应）
6. 游戏区域多边形（控制度着色 + 选中/悬停高亮）
7. 游戏区域标签（图标 + 地名 + 控制度百分比）
8. 恢复裁剪
9. 金色边框
10. 缩放控件按钮（右上角，裁剪区外，始终可见）
```

## 7. 颜色方案

### 7.1 游戏区域 — 控制度着色

| 控制度范围 | 色调 | 含义 |
|-----------|------|------|
| 60~100% | 偏绿色 | 高控制，稳定 |
| 30~59% | 偏橙色 | 中等控制，需巩固 |
| 0~29% | 偏红色 | 低控制，危险 |

选中/悬停时亮度 +15，透明度提升。

### 7.2 背景区域

- 填充：各国独立色调，alpha 40~55（半透明）
- 描边：统一暗金色 rgba(85,75,60,50)
- 标签：rgba(160,148,125)，alpha 随缩放衰减（zoom>3 时开始减淡）

### 7.3 自适应标签

| 缩放级别 | 背景标签 | 游戏区域标签 |
|---------|---------|-------------|
| < 3.0 | 显示（字号 8~12） | 隐藏，只显示圆点标记 |
| 3.0 ~ 4.0 | 渐淡（alpha 下降） | 显示图标 + 地名 |
| ≥ 4.0 | 很淡（alpha ≈ 40） | 显示图标 + 地名 + 控制度% |

## 8. 公开 API

```lua
local MapWidget = require("ui.ui_map_widget")

-- 创建
local map = MapWidget {
    width = "100%",
    height = 420,
    onRegionSelect = function(regionId)
        -- 区域被点击时的回调
    end,
}

-- 设置游戏区域数据（来自 game_state.regions）
map:SetRegions(state.regions)

-- 设置/获取选中的区域 ID
map:SetSelected("capital_city")
local id = map:GetSelected()
```

## 10. 扩展方向

如需扩展地图功能，以下是可能的方向：

- **新增可交互区域**：在 `GAME_REGIONS` 表中添加新的多边形数据，配合 `regions_data.lua` 添加游戏数据
- **势力范围可视化**：根据 AI 势力的 power 值，在对应背景区域叠加半透明颜色层
- **动态边界**：根据战争/外交结果动态修改背景区域的多边形或颜色
- **贸易路线**：在区域之间绘制 NanoVG 曲线表示贸易/军事路线
- **动画效果**：区域脉冲、边界闪烁、战争烟雾等 NanoVG 动画
