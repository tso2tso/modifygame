## 1. 游戏概述

### 1.1 故事背景

1904年，波斯尼亚。科瓦奇家族在巴尔干山区发现了一座金矿，从此开启了一段横跨两次世界大战、四个时代的百年家族传奇。玩家扮演家族的掌舵者，在帝国末日、战争硝烟、工业革命和政治风暴中经营矿业、壮大家族、角逐区域霸权。

### 1.2 游戏时间线

| 章节 | 时代名称 | 年份 | 关键词 | 战争章节 |
|------|---------|------|--------|---------|
| 第一章 | 铜版帝国 | 1904-1913 | 矿业起步、帝国控制 | 否 |
| 第二章 | 战报红章 | 1914-1918 | 第一次世界大战 | **是** |
| 第三章 | 黑金工业 | 1919-1940 | 战后重建、大萧条 | 否 |
| 第四章 | 战时灰幕 | 1941-1945 | 第二次世界大战 | **是** |

- **起始**: 1904年 第1季度
- **终止**: 1945年 第4季度
- **总计**: 约 168 个季度 (42年 × 4季度)

### 1.3 核心玩法

- **矿业经营**: 开采金银煤，升级矿山，管理工人
- **金融投资**: 6只股票的GBM模拟，贷款杠杆
- **科技研发**: 41项科技的分叉互斥树
- **家族管理**: 6个岗位、5维属性匹配
- **区域争霸**: 3个地区的控制度与AI角力
- **历史事件**: 30个固定事件 + 15个随机模板，真实历史节点抉择

## 2. 核心循环与回合结构

### 2.1 回合 = 1个季度

每个回合代表一个季度（春/夏/秋/冬），流程如下：

```
回合开始
  │
  ├─ 阶段0: 通胀推进（影响本季价格）
  ├─ 阶段1: 经济结算（采矿→出售→收入→支出→税收）
  ├─ 阶段1.5: 股市 GBM 更新
  ├─ 阶段1.6: 贷款利息结算 & 到期还本 & 破产检测
  ├─ 阶段1.7: 科技研发推进
  ├─ 阶段2: 事件检查（固定 + 随机同季可并发）
  ├─ 阶段3: 胜利点结算（经济 / 军事）
  ├─ 阶段4: 修正器推进（到期移除）
  ├─ 阶段4.5: 影响力自然衰减（-1/季，文化行动豁免）
  ├─ 阶段5: 武装士气衰减（-2/季，军务主管可减缓）
  ├─ 阶段6: AI 势力更新（增长/扩张/态度变化）
  ├─ 阶段6.5: AI 可能主动进攻
  ├─ 阶段7: 工人士气（工资满足度）
  ├─ 阶段8: 家族培养进度
  └─ 阶段9: 推进季度 → 刷新 AP
```

### 2.2 玩家行动阶段

在结算完成后，玩家进入行动阶段（phase = "action"），可自由分配行动点执行操作，直到点击"结束回合"。

## 5. 经济系统

### 5.1 收入来源

1. **黄金出售**: 手动或自动出售，价格 = 基础价 × 通胀 × 科技加成 × 事件修正
2. **白银出售**: 每季自动全量出售
3. **煤炭出售**: 每季自动全量出售
4. **地区被动收入**:
   - 矿区控制度 ≥50%: 每超10% → +20 × 通胀
   - 工业区控制度 ≥40%: 控制度 × 1.5 × 通胀 × 民需修正
5. **金融网络被动收入**: 科技 `b5_finance_net` 解锁，80/季
6. **贸易被动收入**: 科技 `b4a_trade_route` / `b4b_smuggling` 等
7. **暗影收入**: 事件/修正器带来的灰色收入

### 5.2 支出项目

1. **工人工资**: 工人数 × 基础工资(8) × 劳动力成本因子
2. **军事开支**: 护卫数 × 护卫工资(12) × 通胀
3. **补给开支**: 护卫数 × 3(人均补给) × 2(补给单价) × 通胀
4. **税收**: 现金 × 税率(基础5% / 战时12%)
5. **AI 惩罚**: AI 存在度超50%的地区，每超10% → +15 × 通胀
6. **运输风险损失**: 运输风险修正器带来的收入折损

### 5.3 税率计算

```
有效税率 = 基础税率
  + 事件修正
  - 合法性 × 0.0004
  - 政治地位 × 0.0004
  - 民众支持 × 0.0002
  + 腐败风险 × 0.0005
  + 风险 × 0.0004
  + 监管压力 × 0.0008
  - 首都控制度减税 (≥30%: 每超10% 减1%)
  
范围: 0% ~ 35%
```

```
产出 = (基础 + 事件修正) × 等级乘数 × (1 + 岗位加成) + 工人加成
     × (1 + 科技乘法加成)

- 等级乘数 = 1.0 + (等级-1) × 0.20
- 工人加成 = floor(工人数 / 10 × 工人效率乘数)

```

## 7. 股票市场 (GBM)

### 7.1 模型说明

使用**几何布朗运动 (Geometric Brownian Motion)** 模拟股价:

```
P_{t+1} = P_t × exp((mu - sigma²/2) × dt + sigma × eps × sqrt(dt))
其中 eps ~ N(0,1), dt = 1季度
```

### 7.2 六只股票

| ID | 名称 | 初始价 | 漂移率 mu | 波动率 sigma | 风格 |
|----|------|--------|----------|-------------|------|
| sarajevo_mining | 萨拉热窝矿业 | 12.50 | +0.022 | 0.12 | 稳健成长 |
| imperial_railway | 帝国铁路 | 8.30 | -0.008 | 0.075 | 夕阳衰退 |
| balkan_shipping | 巴尔干航运 | 15.60 | +0.005 | 0.14 | 高波动 |
| military_industry | 军工联合体 | 22.10 | 0.00 | 0.20 | 极高波动 |
| austro_bank_trust | 奥匈银行信托 | 31.40 | +0.014 | 0.09 | 低风险 |
| oriental_trading | 东方贸易公司 | 9.75 | +0.011 | 0.15 | 中等 |

### 7.3 三层修正

1. **L1 基本面**: 每只股票自带的 mu / sigma
2. **L2 事件修正**: `event_mu_mods` 数组，临时叠加 delta_mu
3. **L3 全局修正**: 战争章节 sigma × 1.8 (战时波动放大)

### 7.4 价格保护

- 下限: 1.0
- 上限: 9999.0
- 历史保留: 最近12季度（用于走势图）

### 7.5 买卖机制

- 即时成交，按当前价格计算
- 持仓追踪均价成本
- 卖出即到账

## 8. 贷款与破产

### 8.1 贷款选项（3档）

| 档位 | 额度比 | 最低额 | 基础利率 | 期限(季) |
|------|--------|--------|---------|---------|
| 小额 | 资产×15% | 300 | 4% | 4 |
| 中额 | 资产×35% | 800 | 5% | 6 |
| 大额 | 资产×60% | 2000 | 6% | 8 |

- **最大同时贷款数**: 3
- **最大杠杆率**: 80%
- **动态利率**: 实际利率 = 基础利率 × (1 + 当前杠杆 × 1.5)

### 8.2 违约处理（渐进式）

```
利息到期
  │
  ├─ 现金充足 → 正常付息
  │
  └─ 现金不足 → 强制清算流程:
       ├─ 1. 变卖黄金
       ├─ 2. 降级矿山（回收50%价值）
       ├─ 3. 士气惩罚 -5
       │
       ├─ 清算后凑够 → 正常扣款
       └─ 清算后仍不足 → 真正违约
            └─ 本金膨胀 +15%
```

### 8.3 展期

- 到期还不上本金时允许展期 1 次
- 展期 = 部分清偿 + 延长4季 + 本金膨胀15%

### 8.4 破产条件

| 条件 | 阈值 |
|------|------|
| 连续贷款违约 | 4 季 |
| 净资产持续为负 | 4 季 |
| 预警阈值 | 2 季开始警告 |

### 9.2 影响范围

- `inflation_delta`: 直接修改通胀因子
- `inflation_drift_mod`: 临时修改漂移速率（持续N季）

## 14. 战斗系统

### 14.1 战力计算

**玩家战力**:
```
base = 护卫数 × 1.0(基础战力)
morale = max(0.3, 士气 × 0.01)
equip = 1.0 + (装备等级-1) × 0.15
chief = 1 + 军务主管岗位加成
tech = 1 + 科技战力加成

总战力 = base × morale × equip × chief × tech
```

**AI 战力**:
```
总战力 = 势力值 × 1.0 × (1 + 地区存在度/200)
```

### 14.2 战斗结算

- 双方战力 ×(0.8~1.2) 随机因子
- 玩家 ≥ AI → 玩家胜

### 14.3 胜负结果

**玩家胜利**:
- 缴获 AI 现金 25%
- AI 势力 -8, 态度 -10
- 护卫士气 +10
- 地区: 控制度+3, AI存在度-8

**玩家失败**:
- 折损护卫 30%
- 士气 -18
- 被抢现金 10%
- AI 势力 +5
- 地区: 控制度-5, AI存在度+7, 治安-1

## 16. 事件系统

### 16.1 触发机制

- **固定事件**: 按年份+季度精确触发，不可重复
- **随机事件**: 每季检查触发条件 + 概率判定
- **并发**: 固定和随机可同季触发（有固定时随机最多1个，否则最多2个）

### 16.2 保底机制

| 连续无事件季数 | 概率乘数 |
|---------------|---------|
| 0-1 季 | ×1.0 |
| 2 季 | ×1.5 |
| ≥3 季 | ×2.0 |

### 16.3 随机事件触发条件

支持以下条件组合:
- `requires_mine`: 需要有活跃矿山
- `max_security`: 治安上限
- `min_workers`: 最低工人数
- `min_year` / `max_year`: 年份范围
- `requires_war`: 需要战时
- `min_development`: 最低基建
- `cooldown`: 冷却季度数
- `chance`: 基础概率

### 16.4 效果类型

- **直接资源**: cash, gold, gold_reserve, workers_bonus, security_bonus
- **宏观冲击**: inflation_delta, inflation_drift_mod, asset_price_mod, war_state
- **修正器**: 持续效果 (target + value + duration)
- **股市联动**: 通过 event_market_effects 注入 delta_mu
- **AP 奖励**: bonus_ap

## 20. 胜利与失败条件

### 20.1 经济胜利

| 条件 | 值 |
|------|-----|
| 胜利点阈值 | 2000 |
| 门控年份 | ≥1930年 |
| **快照验证** | |
| 最低现金 | 15,000 |
| 最低黄金 | 25 |
| 最低总控制 | 100 |

**经济胜利点每季计算**:
```
delta = floor(现金/3000) + floor(黄金×0.35) + floor(总控制/20) + floor(总影响/60)
战时: delta × 0.6
影响力≥300: +5
```

### 20.2 军事胜利

| 条件 | 值 |
|------|-----|
| 胜利点阈值 | 2500 |
| 门控年份 | ≥1925年 |
| **快照验证** | |
| 最低护卫 | 30 |
| 最低士气 | 55 |
| 最低总控制 | 120 |

**军事胜利点每季计算**:
```
delta = floor(护卫×0.25) + floor(士气/28) + floor(总控制/14) + min(未结算胜场, 3)
战时: delta × 1.25
影响力≥300: +5
```
