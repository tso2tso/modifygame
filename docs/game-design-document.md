## 1. 游戏概述

### 1.1 故事背景

1904年，波斯尼亚。科瓦奇家族在巴尔干山区发现了一座金矿，从此开启了一段横跨两次世界大战、五个时代的半世纪家族传奇。玩家扮演家族的掌舵者，在帝国末日、战争硝烟、工业革命、政治风暴和战后重建中经营矿业、壮大家族、角逐区域霸权。

### 1.2 游戏时间线

| 章节 | 时代名称 | 年份 | 关键词 | 战争章节 |
|------|---------|------|--------|---------|
| 第一章 | 铜版帝国 | 1904-1913 | 矿业起步、帝国控制 | 否 |
| 第二章 | 战报红章 | 1914-1918 | 第一次世界大战 | **是** |
| 第三章 | 黑金工业 | 1919-1940 | 战后重建、大萧条 | 否 |
| 第四章 | 战时灰幕 | 1941-1945 | 第二次世界大战 | **是** |
| 第五章 | 战后余烬 | 1946-1955 | 社会主义改造、冷战博弈 | 否 |

- **起始**: 1904年 第1季度
- **终止**: 1955年 第4季度
- **总计**: 约 204 个季度 (51年 × 4季度)

### 1.3 核心玩法

- **矿业经营**: 开采金银煤，升级矿山，管理工人
- **金融投资**: 6只股票的GBM模拟，贷款杠杆
- **科技研发**: 37项科技的分叉互斥树
- **家族管理**: 6个岗位、5维属性匹配
- **区域争霸**: 3个地区的控制度与AI角力
- **历史事件**: 43个固定事件 + 14个随机模板 + 5个分支事件，真实历史节点抉择
- **大国博弈**: 15个大国的军事/经济/战争疲劳模拟，征服与继承
- **欧洲地图**: 17个国家的主权变迁、抵抗运动与解放

## 2. 核心循环与回合结构

### 2.1 回合 = 1个季度

每个回合代表一个季度（春/夏/秋/冬），流程如下：

```
回合开始
  │
  ├─ 阶段0: 通胀推进（影响本季价格）
  ├─ 阶段1: 经济结算（采矿→出售→收入→支出→税收）
  ├─ 阶段1.5: 股市 GBM 更新
  ├─ 阶段1.6: 贷款利息结算 & 到期还本 & 破产检测
  ├─ 阶段1.7: 科技研发推进
  ├─ 阶段2: 事件检查（固定 + 随机同季可并发）
  ├─ 阶段3: 胜利点结算（经济 / 军事）
  ├─ 阶段4: 修正器推进（到期移除）
  ├─ 阶段4.5: 影响力自然衰减（-1/季，文化行动豁免）
  ├─ 阶段5: 武装士气衰减（-2/季，军务主管可减缓）
  ├─ 阶段6: AI 势力更新（增长/扩张/态度变化/主动消费）
  ├─ 阶段6.2: 外资控制度持续性修正器（基于全局外资存在度）
  ├─ 阶段6.5: AI 可能主动进攻
  ├─ 阶段6.7: 大国博弈（激活/漂移/派系/继承/征服/抵抗/代理AI）
  ├─ 阶段6.8: 历史分支事件检查（5个关键分支决策点）
  ├─ 阶段7: 工人士气（工资满足度）
  ├─ 阶段8: 家族培养进度
  └─ 阶段9: 推进季度 → 刷新 AP
```

### 2.2 玩家行动阶段

在结算完成后，玩家进入行动阶段（phase = "action"），可自由分配行动点执行操作，直到点击"结束回合"。

## 5. 经济系统

### 5.1 收入来源

1. **黄金出售**: 手动或自动出售，价格 = 基础价 × 通胀 × 科技加成 × 事件修正
2. **白银出售**: 每季自动全量出售
3. **煤炭采集与出售**: 工业区有煤炭储量时，每季自动采集（基础8单位）并全量出售。**采煤无控制度门槛**，只要工业区有煤炭储量即可采集。
4. **地区被动收入**:
   - 矿区控制度 ≥50%: 每超10% → +20 × 通胀
   - 工业区控制度 ≥40%: 控制度 × 1.5 × 通胀 × 民需修正（此为**贸易收入**，与煤炭采集独立）
   > ⚠️ 注意：煤炭采集收入和工业区贸易收入是两个独立来源。玩家初始工业区控制度20%，低于40%贸易门槛，但仍可获得煤炭采集收入。
5. **金融网络被动收入**: 科技 `b5_finance_net` 解锁，80/季
6. **贸易被动收入**: 科技 `b4a_trade_route` / `b4b_smuggling` 等
7. **暗影收入**: 事件/修正器带来的灰色收入

### 5.2 支出项目

1. **工人工资**: 工人数 × 基础工资(8) × 劳动力成本因子
2. **军事开支**: 护卫数 × 护卫工资(12) × 通胀
3. **补给开支**: 护卫数 × 3(人均补给) × 2(补给单价) × 通胀
4. **税收（财富税）**: 当前持有现金 × 税率(基础5% / 战时12%)
   > ⚠️ 注意：这是对**现金存量**征税，而非对当季收入征税。持有现金越多税额越高，形成主动花钱的逆向激励。达到胜利快照要求的15,000现金时，季度税额为750（战时1,800）。
5. **AI 惩罚**: AI 存在度超50%的地区，每超10% → +15 × 通胀
6. **运输风险损失**: 运输风险修正器带来的收入折损

### 5.3 税率计算

```
有效税率 = 基础税率
  + 事件修正
  - 合法性 × 0.0004
  - 政治地位 × 0.0004
  - 民众支持 × 0.0002
  + 腐败风险 × 0.0005
  + 风险 × 0.0004
  + 监管压力 × 0.0008
  - 首都控制度减税 (≥30%: 每超10% 减1%)
  
范围: 0% ~ 35%
```

## 7. 股票市场 (GBM)

### 7.1 模型说明

使用**几何布朗运动 (Geometric Brownian Motion)** 模拟股价:

```
P_{t+1} = P_t × exp((mu - sigma²/2) × dt + sigma × eps × sqrt(dt))
其中 eps ~ N(0,1), dt = 1季度
```

### 7.2 六只股票

| ID | 名称 | 初始价 | 漂移率 mu | 波动率 sigma | 风格 |
|----|------|--------|----------|-------------|------|
| sarajevo_mining | 萨拉热窝矿业 | 12.50 | +0.022 | 0.12 | 稳健成长 |
| imperial_railway | 帝国铁路 | 8.30 | -0.008 | 0.075 | 夕阳衰退 |
| balkan_shipping | 巴尔干航运 | 15.60 | +0.005 | 0.14 | 高波动 |
| military_industry | 军工联合体 | 22.10 | 0.00 | 0.20 | 极高波动 |
| austro_bank_trust | 奥匈银行信托 | 31.40 | +0.014 | 0.09 | 低风险 |
| oriental_trading | 东方贸易公司 | 9.75 | +0.011 | 0.15 | 中等 |

### 7.3 三层修正

1. **L1 基本面**: 每只股票自带的 mu / sigma
2. **L2 事件修正**: `event_mu_mods` 数组，临时叠加 delta_mu
3. **L3 全局修正**: 战争章节 sigma × 1.8 (战时波动放大)

### 7.4 价格保护

- 下限: 1.0
- 上限: 9999.0
- 历史保留: 最近12季度（用于走势图）

### 7.5 买卖机制

- 即时成交，按当前价格计算
- 持仓追踪均价成本
- 卖出即到账

## 8. 贷款与破产

### 8.1 贷款选项（3档）

| 档位 | 额度比 | 最低额 | 基础利率 | 期限(季) |
|------|--------|--------|---------|---------|
| 小额 | 资产×15% | 300 | 4% | 4 |
| 中额 | 资产×35% | 800 | 5% | 6 |
| 大额 | 资产×60% | 2000 | 6% | 8 |

- **最大同时贷款数**: 3
- **最大杠杆率**: 80%
- **动态利率**: 实际利率 = 基础利率 × (1 + 当前杠杆 × 1.5)

### 8.2 违约处理（渐进式）

```
利息到期
  │
  ├─ 现金充足 → 正常付息
  │
  └─ 现金不足 → 强制清算流程:
       ├─ 1. 变卖黄金
       ├─ 2. 降级矿山（回收50%价值）
       ├─ 3. 士气惩罚 -5
       │
       ├─ 清算后凑够 → 正常扣款
       └─ 清算后仍不足 → 真正违约
            └─ 本金膨胀 +15%
```

### 8.3 展期

- 到期还不上本金时允许展期 1 次
- 展期 = 部分清偿 + 延长4季 + 本金膨胀15%

### 8.4 破产条件

| 条件 | 阈值 |
|------|------|
| 连续贷款违约 | 4 季 |
| 净资产持续为负 | 4 季 |
| 预警阈值 | 2 季开始警告 |

## 14. 战斗系统

### 14.1 战力计算

**玩家战力**:
```
base = 护卫数 × 1.0(基础战力)
morale = max(0.3, 士气 × 0.01)
equip = 1.0 + (装备等级-1) × 0.15
chief = 1 + 军务主管岗位加成
tech = 1 + 科技战力加成

总战力 = base × morale × equip × chief × tech
```

**护卫士气参数**:
| 参数 | 值 |
|------|-----|
| 初始值 | 70 |
| 范围 | 0 ~ 100 |
| 每季自然衰减 | -2（军务主管可减缓） |
| 胜利提升 | +10 |
| 失败损失 | -18 |
| 战力影响 | max(0.3, 士气 × 0.01)，即士气70→×0.7 |

**AI 战力**:
```
总战力 = 势力值 × 1.0 × (1 + 地区存在度/200)
```

### 14.2 战斗结算

- 双方战力 ×(0.8~1.2) 随机因子
- 玩家 ≥ AI → 玩家胜

### 14.3 胜负结果

**玩家胜利**:
- 缴获 AI 现金 25%
- AI 势力 -8, 态度 -10
- 护卫士气 +10
- 地区: 控制度+3, AI存在度-8

**玩家失败**:
- 折损护卫 30%
- 士气 -18
- 被抢现金 10%
- AI 势力 +5
- 地区: 控制度-5, AI存在度+7, 治安-1

## 16. 事件系统

### 16.1 触发机制

- **固定事件**: 按年份+季度精确触发，不可重复
- **随机事件**: 每季检查触发条件 + 概率判定
- **并发**: 固定和随机可同季触发（有固定时随机最多1个，否则最多2个）

### 16.2 保底机制

| 连续无事件季数 | 概率乘数 |
|---------------|---------|
| 0-1 季 | ×1.0 |
| 2 季 | ×1.5 |
| ≥3 季 | ×2.0 |

### 16.3 随机事件触发条件

支持以下条件组合:
- `requires_mine`: 需要有活跃矿山
- `max_security`: 治安上限
- `min_workers`: 最低工人数
- `min_year` / `max_year`: 年份范围
- `requires_war`: 需要战时
- `min_development`: 最低基建
- `cooldown`: 冷却季度数
- `chance`: 基础概率

### 16.4 效果类型

- **直接资源**: cash, gold, gold_reserve, workers_bonus, security_bonus
- **宏观冲击**: inflation_delta, inflation_drift_mod, asset_price_mod, war_state
- **修正器**: 持续效果 (target + value + duration)
- **股市联动**: 通过 event_market_effects 注入 delta_mu
- **AP 奖励**: bonus_ap

## 20. 胜利与失败条件

### 20.1 经济胜利 (v3)

| 条件 | 值 | v2→v3 变化 |
|------|-----|-----------|
| 胜利点阈值 | **1,600** | 2000→1600 |
| 门控年份 | ≥1930年 | 不变 |
| **快照验证** | | |
| 最低现金 | **10,000** | 15000→10000 |
| 最低黄金 | **20** | 25→20 |
| 最低总控制 | **80** | 100→80 |

**经济胜利点每季计算**:
```
delta = floor(现金/2000) + floor(黄金×0.5) + floor(总控制/15) + floor(总影响/50)
战时: delta × 0.6
影响力≥300: +5
```

> **v3 调整说明**: 时间线延长至1955年后，整体降低了阈值和快照门槛，同时提高了公式系数（现金除数3000→2000，黄金乘数0.35→0.5，控制除数20→15，影响除数60→50），使经济胜利在1945-1950年间可达。

### 20.2 军事胜利 (v3)

| 条件 | 值 | v2→v3 变化 |
|------|-----|-----------|
| 胜利点阈值 | **2,000** | 2500→2000 |
| 门控年份 | ≥1925年 | 不变 |
| **快照验证** | | |
| 最低护卫 | **25** | 30→25 |
| 最低士气 | **50** | 55→50 |
| 最低总控制 | **100** | 120→100 |

**军事胜利点每季计算**:
```
delta = floor(护卫×0.3) + floor(士气/25) + floor(总控制/12) + min(未结算胜场, 3)
战时: delta × 1.25
影响力≥300: +5
```

> **v3 调整说明**: 军事线从2500→2000缩小了与经济线的难度差距，护卫乘数0.25→0.3、士气除数28→25、控制除数14→12，让护卫投资和战斗获胜更有价值。在1940-1950年间可达。

```
DRIFT_RATE = 0.10
```

## 27. 玩家-大国行动与合作指数

### 27.1 概述

玩家可以对当前活跃的大国执行9种行动，分为4种立场。每种行动消耗AP和现金，并累加或扣减**合作指数（collaboration_score）**。合作指数决定战后清算结局（§26.4）。

### 27.2 四种立场

| 立场 | 含义 | 合作指数方向 |
|------|------|-------------|
| collaborate（合作） | 与大国合作获利 | ↑ 增加 |
| join（加入） | 直接参与大国行动 | ↑↑ 大幅增加 |
| counter（反制） | 经济手段对抗大国 | ↓ 减少 |
| resist（抵抗） | 直接抵抗大国 | ↓↓ 大幅减少 |

### 27.3 九种行动

#### 合作立场 (collaborate)

| 行动 | AP | 费用 | 合作指数 | 效果 |
|------|-----|------|---------|------|
| 战争物资供应 (war_supplier) | 1 | 现金 | +5 | 向大国出售物资获利 |
| 合作管理 (cooperative_management) | 1 | 现金 | +3 | 与大国共同管理地区事务 |

#### 加入立场 (join)

| 行动 | AP | 费用 | 合作指数 | 效果 |
|------|-----|------|---------|------|
| 派遣志愿军 (send_volunteers) | 2 | 现金 | +8 | 派人参加大国军事行动 |
| 情报共享 (share_intel) | 1 | 现金 | +5 | 与大国共享情报 |

#### 反制立场 (counter)

| 行动 | AP | 费用 | 合作指数 | 效果 |
|------|-----|------|---------|------|
| 经济制裁 (economic_sanction) | 2 | 现金 | -3 | 对大国实施贸易限制 |
| 货币战争 (currency_war) | 2 | 现金 | -5 | 通过金融手段对抗大国 |

#### 抵抗立场 (resist)

| 行动 | AP | 费用 | 合作指数 | 效果 |
|------|-----|------|---------|------|
| 支援游击队 (support_guerrilla) | 2 | 现金 | -8 | 资助被占领国抵抗力量 |
| 庇护难民 (shelter_refugees) | 1 | 现金 | -5 | 收容战争难民 |
| 破坏补给 (sabotage_supply) | 2 | 现金 | -10 | 破坏大国后勤补给线 |

### 27.4 合作指数

合作指数是一个跨越整个游戏的累加值，记录玩家与大国交往的历史倾向：


## 附录: 数值平衡备注

> 以下为对现有数值体系的分析，标记可能需要调整的设计点。

### B1. 经济胜利可达性分析（v3）

- **门控年份**: 1930年（距游戏结束1955Q4共100季）
- **v3公式**: `floor(cash/2000) + floor(gold×0.5) + floor(control/15) + floor(influence/50)`
- **中等状态模拟** (现金10000, 黄金25, 控制100, 影响200):
  - 每季增量: floor(10000/2000)+floor(25×0.5)+floor(100/15)+floor(200/50) = 5+12+6+4 = **27点/季**
  - 和平期 1930-1940 (44季): 27×44 = 1188
  - 战时 1941-1945 (20季): 27×0.6≈16, 16×20 = 320
  - 战后 1946-1955 (36季): 27×36 = 972
  - **合计约2480，超过阈值1600** ✅
- **v2→v3 变化**: 阈值2000→1600，系数全面放宽，时间线延长40季。v2下"几乎不可能"的经济胜利变为"中等难度可达成"。
- **结论**: v3经济胜利在中等状态下可达，但仍需稳定经营。早期破产或战时重创可导致积分断档。快照要求（现金≥10000, 黄金≥20, 控制≥80）是主要门槛。

### B2. 军事胜利可达性分析（v3）

- **门控年份**: 1925年（距游戏结束1955Q4共120季）
- **v3公式**: `floor(guards×0.3) + floor(morale/25) + floor(control/12) + battle_wins`
- **中等状态模拟** (护卫30, 士气60, 控制100, 每季1胜):
  - 每季增量: floor(30×0.3)+floor(60/25)+floor(100/12)+1 = 9+2+8+1 = **20点/季**
  - 和平期 1925-1940 (64季): 20×64 = 1280
  - 战时 1941-1945 (20季): 20×1.25=25, 25×20 = 500
  - 战后 1946-1955 (36季): 20×36 = 720
  - **合计约2500，超过阈值2000** ✅
- **v2→v3 变化**: 阈值2500→2000，系数放宽（guard×0.25→0.3, morale/28→/25, control/14→/12），时间线延长40季。
- **结论**: v3军事胜利在中等状态下刚好可达。相比经济路线，军事路线的优势在于战时增益（×1.25），但维持30护卫每季固定支出约540克朗，经济压力仍显著。快照要求（护卫≥25, 士气≥50, 控制≥100）低于v2，降低了准入门槛。

### B3. AI外资增长速度（已修复）

- **维也纳矿业**: 初始现金2000，每季增长8%，**现金上限12000**
- 无上限时: 2000 × 1.08⁴⁰ ≈ 43,450克朗（v2问题：中后期AI实力远超玩家干预能力）
- **v3修复**: 增加 `cash_cap`（帝国8000, 维也纳矿业12000），复利增长在达到上限后停止
  - 维也纳矿业: 2000 → 12000 约需24季（约6年），之后现金锁定在12000
  - 帝国银行: 3000 → 8000 约需13季（约3年），之后现金锁定在8000
- **AI主动消费**: 达到上限后AI通过消费行为（雇佣兵、制裁、通胀操纵等）花费现金，形成"赚→花→赚"循环，避免资金空转
- **结论**: cash_cap 彻底解决了v2的AI失控问题，同时消费行为保持了AI的威胁感

### B4. 广告奖励与早期经济

- **期望值**: 加权平均 ≈ 883克朗/次（含衰减后三次合计约1934克朗/季）
- **早期净收入**: 约200-400克朗/季
- **问题**: 单季广告收入可达早期净收入的5-10倍，严重稀释早期经济压力
- **建议**: 考虑早期奖励缩放（如乘以 min(1, 年份进度/0.3)），或降低前10年的基础奖金。

### B5. 财富税的逆向激励

- 税率5%对现金存量征收 → 持有10000现金时季度税额500，战时1200
- 玩家倾向于将现金立即转化为黄金/股票/矿山升级，而非储蓄
- 这与胜利快照要求"最低现金10000"形成矛盾：积攒现金会被重税消耗
- **v3缓解**: 快照最低现金从15000降至10000，降低了囤现金的税收惩罚（税额500 vs 750）
- **建议**: 在UI中添加"胜利条件进度"提示，让玩家清楚权衡
