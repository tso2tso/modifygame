# 百年萨拉热窝：黄金家族 — 主界面 UI 描述文档

> **版本**: v1.0
> **最后更新**: 2026-04-26
> **对应代码**: `scripts/ui/` 目录

---

## 1. 整体架构

### 1.1 页面层级

```
SafeAreaView (gameRoot, 全屏)
├── TopBar（顶栏，双行，固定顶部）
│   ├── Row1: 信息栏（年份 + 资源）    72px
│   └── Row2: 行动点栏（AP + 安全等级） 52px
├── ContentArea（内容区，flex-grow 填充中部）
│   ├── Dashboard（仪表盘 / 主视图，ScrollView）
│   ├── FamilyPage（家族页，隐藏态）
│   ├── IndustryPage（产业页，隐藏态）
│   ├── MarketPage（市场页，隐藏态）
│   ├── MilitaryPage（武装页，隐藏态）
│   └── WorldPage（世界页，隐藏态）
├── BottomNav（底部导航栏）            64px
└── [Drawer]（右侧抽屉 - 设置，按需弹出）
```

### 1.2 视觉风格

- **设计语言**：工业帝国主义时代的家族账簿
- **配色基调**：深色背景（`#1A1610` bg_base）+ 羊皮纸色文字（`#F0E6D0` text_primary）
- **强调色**：金色（`#C9A84C` accent_gold），随时代章节变化
- **竖屏设计**：面向手机竖屏（portrait），所有布局纵向优先
- **字体**：MiSans-Regular，统一 sans 字族

### 1.3 屏幕适配

- 使用 `UI.Scale.DEFAULT` 缩放策略
- 通过 `graphics:GetWidth() / GetHeight() / GetDPR()` 获取屏幕信息
- `SetMode()` 已禁用，不设置固定分辨率

## 2. TopBar 顶栏（ui_top_bar.lua）

顶栏固定在屏幕最顶部，共两行，总高度 **124px**（72 + 52），提供全局状态总览。

### 2.1 Row1: 信息栏（72px）

**布局**：水平排列（flexDirection="row"），左中右三区

```
┌─────────────────────────────────────────────────────────────┐
│ 1904年 Q1 春季  💰1,000 🎰 │ 🟡5    │ ⛏50   │ ⭐0   │ 🏦净资产 │
│ 4月15日          现金        │ 黄金(吨)│ 产能   │ 声望  │          │
│ 第一章 铜版帝国              │         │        │       │          │
└─────────────────────────────────────────────────────────────┘
```

| 区域 | 内容 | 样式 | 说明 |
|------|------|------|------|
| **左侧 - 时间区** | 年份+季度+季节名 | 22px bold，era.accent 颜色 | 如 "1904年 Q1 春季" |
| | 具体日期 | 12px，text_secondary | 如 "4月15日" |
| | 时代标签 | 10px，era.accent 半透明 | 如 "第一章 铜版帝国" |
| **中部 - 资源区** | 4组核心资源 + 竖线分隔 | icon+数值16px bold+标签10px | 竖线 1px paper_mid |
| 资源1 | 💰 现金 | era.accent 颜色 | 当前克朗余额，FormatNumber 格式化 |
| 广告按钮 | 🎰 | 26×26 圆形，淡金底+金边 | 看激励广告获随机克朗奖励 |
| 资源2 | 🟡 黄金(吨) | text_primary | 当前黄金库存 |
| 资源3 | ⛏️ 产能 | text_primary | `mine.level * 30 + workers.hired * 2` |
| 资源4 | ⭐ 声望 | text_primary | 首个区域的 influence 值 |
| **右侧 - 净资产** | 🏦 净资产 | 正数绿色/负数红色 | `总资产 - 总负债` |

**特殊效果**：
- 战争时代（第2/4/6章）叠加 8% 暗红色遮罩（`warStripe`）
- 底部 1px 边框线使用 era.accent 30% 透明

### 2.2 Row2: 行动点栏（52px）

**布局**：水平排列，bg_surface 背景

```
┌────────────────────────────────────────────────────────┐
│ (AP) 5 / 5  ●●●●●  [+]            安全等级           │
│      本季可用行动点                 🛡 勉强维持        │
└────────────────────────────────────────────────────────┘
```

| 区域 | 内容 | 样式 | 说明 |
|------|------|------|------|
| **AP 图标** | 圆形 "AP" 标签 | 32×32，paper_dark 底，era.accent 边框+文字 | 固定不变 |
| **AP 数值** | `已用/总量` | 18px bold + 11px 副标题 | 如 "5 / 5" + "本季可用行动点" |
| **圆点进度条** | ● 实心 / ○ 空心 | 8px 圆点，era.accent 填充 | 临时AP用半透明色区分 |
| **[+] 按钮** | 购买额外AP | 36×36，paper_dark 底，era.accent 边框 | 花费现金换1临时AP，每季有上限 |
| **安全等级** | 🛡️ + 等级文字 | 胶囊标签，背景色按等级变化 | 红(≤2)/橙(3)/绿(≥4) |

**安全等级颜色规则**：
- ≤2：`accent_red` 背景（危险）
- 3：`accent_amber` 背景（一般）
- ≥4：`accent_green` 背景（安全）

## 3. ContentArea 内容区

内容区占据顶栏和底部导航之间的全部剩余空间。采用 **绝对定位叠层**（position="absolute"），通过 `SetVisible` 切换当前可见页面。

### 3.1 默认视图：Dashboard 仪表盘（ui_dashboard.lua）

仪表盘是游戏的主视图（首页），外层为 ScrollView，内部纵向排列以下区块：

#### 3.1.1 本季动态（Turn Messages）

**显示条件**：`state.turn_messages` 非空时才显示

```
┌──────────────────────────────────────┐
│ 本季动态（2）              [通知]    │
│ ┌──────────────────────────────────┐ │
│ │ ⚔ 矿区护卫队击退了土匪袭击     │ │
│ ├──────────────────────────────────┤ │
│ │ 📢 奥匈当局增加了矿税          │ │
│ └──────────────────────────────────┘ │
└──────────────────────────────────────┘
```

| 消息类型 | 图标 | 背景色 | 说明 |
|---------|------|--------|------|
| `combat_win` | ⚔ | 深绿 `{45,100,55}` | 战斗胜利 |
| `combat_lose` | 💥 | 深红 `{120,35,35}` | 战斗失败 |
| `ai_move` | 📢 | 深棕 `{60,55,45}` | AI行动通知 |
| `warning` | ⚠ | 深橙 `{110,85,30}` | 经济警告 |

#### 3.1.2 事件流（Event Stream）

显示当前待处理的游戏事件队列。

```
┌──────────────────────────────────────┐
│ 当前事件（1）              [待处理]  │
│ ┌──────────────────────────────────┐ │
│ │ ┃ [📜]  ❗主线 金矿矿权  [处理] │ │
│ │ ┃       经过多年在Baković矿区... │ │
│ └──────────────────────────────────┘ │
└──────────────────────────────────────┘
```

**单个事件卡片结构**（水平三栏）：

| 区域 | 内容 | 说明 |
|------|------|------|
| **左侧** | 64×64 图片区（emoji 图标占位） | 如 📜⛏️🏛️ |
| **中间** | 优先级徽章 + 标题 + 描述(最多2行) + 剩余时间 | 主线：红底白字徽章；支线：paper_mid 底 |
| **右侧** | "处理" 按钮 80×32 | paper_dark 底 + era.accent 边框文字 |

**优先级样式**：
- **主线事件**：左侧 3px era.accent 竖边框 + ❗主线红底徽章 + 标题 accent_red
- **支线事件**：无竖边框 + "支线" paper_mid 底徽章 + 标题 text_primary

**空状态**：居中显示 "暂无待处理事件"

#### 3.1.3 经营焦点卡片（Focus Card）

展示第一个矿山的详细运营数据，类似"仪表板重点关注项"。

```
┌──────────────────────────────────────┐
│ 科瓦奇金矿  [金矿开采]        ...   │
│ ┌────────┬───────────────────────┐   │
│ │  ⛏️    │ 产量(本季)  产能利用率│   │
│ │  图片  │ 2 单位      100%     │   │
│ │        │ 工人状态    维护费用  │   │
│ │        │ 😐稳定      -80      │   │
│ └────────┴───────────────────────┘   │
│ 📦库存    💎品质   💰开采成本  🛡安全 │
│ 5 单位    低       50/单位    勉强维持│
│ ─────────────────────────────────── │
│ [雇佣工人(1 AP)]  [升级设施(2 AP)]   │
└──────────────────────────────────────┘
```

**卡片分区**：

| 区域 | 内容 | 说明 |
|------|------|------|
| **头部** | 矿山名 + "金矿开采"类型胶囊 + "…"更多 | bg_elevated 胶囊 |
| **核心KPI区** | 左图片(100×100) + 右2×2指标网格 | 产量、利用率(带进度条)、工人状态、维护费用 |
| **次要指标** | 4格横排小卡片 | 库存、品质、开采成本、安全等级(彩色胶囊) |
| **操作按钮** | 2个等宽按钮 | 雇佣工人(1AP) + 升级设施(2AP，右上角amber徽章"2") |

**利用率进度条颜色**：≥80% 绿 / 50-79% 橙 / <50% 红

**操作按钮功能**：
- **雇佣工人**（1 AP）：花费 `hire_cost × 劳动力因子 × (1-影响力折扣) × 5`，招募5名工人
- **升级设施**（2 AP）：花费 `upgrade_cost × 当前等级 × 资产价格因子`，矿山等级+1

## 4. BottomNav 底部导航栏（ui_manager.lua）

固定在屏幕底部，高度 **64px**。

```
┌──────┬──────┬──────┬──────┬──────┐
│  👥  │  ⛏️  │  📈  │  🛡️  │  🌍  │
│ 家族 │ 产业 │ 市场 │ 武装 │ 世界 │
└──────┴──────┴──────┴──────┴──────┘
```

| Tab | 图标 | 页面 | 功能概述 |
|-----|------|------|---------|
| 家族 | 👥 | ui_family.lua | 家族成员管理、岗位分配、属性查看 |
| 产业 | ⛏️ | ui_industry.lua | 矿山详情、工人管理、收支预估、操作按钮 |
| 市场 | 📈 | ui_market.lua | 股票/债券/商品交易、贷款操作 |
| 武装 | 🛡️ | ui_military.lua | 护矿队管理、士气、装备、招募 |

**选中态样式**：
- 顶部 2px era.accent 指示线
- 图标+文字使用 era.accent 颜色
- 背景 bg_elevated

**未选中态样式**：
- 图标+文字使用 tab_inactive 颜色
- 无顶部线条
- 背景 bg_base

**导航逻辑**：
- 点击未激活的 Tab → 切换到对应深度页
- 再次点击已激活的 Tab → 返回仪表盘（主视图）
- 顶部 1px paper_light 分隔线

## 5. 深度页面概述

所有深度页面共享：ScrollView 容器 + page_padding(12px) 内边距 + 纵向 card_gap(10px) 间距。

### 5.1 家族页（ui_family.lua）

**功能**：家族成员管理、岗位分配

**核心组件**：
- **家族概况卡片**：家族人数、当前年份、总体状态
- **成员卡片列表**：每个成员展示姓名、年龄、5维属性条（管理/谋略/魅力/学识/野心）
- **岗位分配**：通过 `UI.Dropdown` 为每个成员分配岗位（矿业总监、军务主管、财务总监、文化顾问、科技顾问、外交总监）
- **岗位适配度标签**：满配/半配/差配

### 5.2 产业页（ui_industry.lua）

**功能**：矿山运营详情、工人管理

**核心组件**：
- **收支预估卡片**：本季预估收入/支出/利润
- **矿山卡片**（每座矿山一张）：等级、产量、储量、升级/开发操作
- **工人管理卡片**：在岗人数、工资、士气、招聘/解雇操作
- **操作按钮区**：探矿、新建矿山等高级操作

### 5.3 市场页（ui_market.lua）

**功能**：金融市场交易

**核心组件**：
- **子标签页**：股票 / 债券 / 商品 / 贷款
- **股票板块**：GBM 动态股价 + 买卖交互 + 微型走势图（矿业/运输/军工/金融/贸易）
- **贷款操作**：申请贷款、查看已有贷款、还款
- **黄金交易**：黄金出售

### 5.4 武装页（ui_military.lua）

**功能**：护矿队管理

**核心组件**：
- **军事概况**：护卫人数、士气、战力、每季军费
- **军务主管信息**：当前指派的家族成员 + 加成效果
- **操作**：招募护卫、提升士气、购买装备

### 5.5 世界页（ui_world.lua）

**功能**：世界局势总览

**核心组件**：
- **子 Tab 系统**：地图 | 关系 | 势力 | 报告
- **地图标签**：节点式地图（MapWidget），显示各区域控制状态
- **关系标签**：与各势力的外交关系
- **势力标签**：各方势力实力对比
- **报告标签**：历史事件报告
- **节点信息抽屉**：点击地图节点后从侧边弹出详情

## 6. 弹窗系统

### 6.1 操作弹窗（ui_action_modals.lua）

由快速操作区触发，以 Modal 形式覆盖在主 UI 之上。

| 弹窗 | 触发 | 功能 |
|------|------|------|
| 科技研发 | 快速操作 "科技研发" | 科技树、研究项目 |
| 情报行动 | 快速操作 "情报行动" | 侦察、破坏等情报任务 |
| 政治外交 | 快速操作 "政治外交" | 与势力的外交互动 |
| 资产交易 | 快速操作 "资产交易" | 矿山买卖等资产操作 |

### 6.2 事件处理弹窗（ui_event_modal.lua）

点击事件卡片的"处理"按钮后弹出，展示事件详情和选项。

### 6.3 结局弹窗（ui_ending_modal.lua）

游戏结束时显示，展示最终成就和统计数据，提供"新游戏"选项。

### 6.4 设置抽屉（Drawer）

从屏幕右侧滑入，宽度约 58%。

**内容**（ui_menu.lua）：
- 游戏设置选项
- 存档/读档
- 新游戏
- 左侧 1px era.accent 竖边框 + 左上/左下 8px 圆角

### 6.5 Toast 通知

全局轻量提示，固定宽度 320×56px。

| variant | 用途 | 示例 |
|---------|------|------|
| `success` | 操作成功 | "+1 AP"、"招募 +5 工人" |

## 8. 时代系统（Era System）

游戏跨越 1904-2014 年，分 7 个时代章节，每个时代有独立的 accent 颜色：

| 章节 | 年份范围 | 名称 | 特征 |
|------|---------|------|------|
| 第一章 | 1904-1914 | 铜版帝国 | 起步阶段 |
| 第二章 | 1914-1918 | 战报红章 | 一战，战争遮罩 |
| 第三章 | 1918-1938 | 黑金工业 | 工业扩张 |
| 第四章 | 1938-1945 | 战时灰幕 | 二战，战争遮罩 |
| 第五章 | 1945-1980 | 制度蓝红 | 冷战时期 |
| 第六章 | 1980-1995 | 围城信号 | 波黑战争，战争遮罩 |
| 第七章 | 1995-2014 | 玻璃金融 | 现代金融 |

每个时代的 `accent` 颜色会影响：
- 顶栏年份文字颜色
- 现金数值颜色
- AP 圆点填充色 + [+]按钮边框
- 事件卡片"处理"按钮边框
- 底部导航选中态指示线+文字
- 结束回合按钮填充色
- 设置抽屉边框

## 9. 交互流程

### 9.1 核心游戏循环

```
仪表盘（查看状态）
  ↓
处理事件 → 事件弹窗 → 选择选项 → 返回仪表盘
  ↓
快速操作 → 弹窗/导航 → 执行操作 → 返回仪表盘
  ↓
焦点卡片操作 → 雇佣/升级 → 状态刷新
  ↓
结束回合 → 结算（经济/战斗/事件生成）→ 下一季
```

### 9.2 导航流

```
仪表盘 ←→ 深度页（底部Tab切换）
         ↑ 再次点击同Tab返回仪表盘
仪表盘 → 设置抽屉（右侧滑出）
```

### 9.3 触控适配

- 所有按钮 ≥ 36px 触控区域
- TapGuard 防误触：按下计数器 + 滑动距离过滤（20px 阈值）
- 移动端抑制模拟鼠标事件（仅保留触摸路径），避免 Dropdown 等组件双触发
