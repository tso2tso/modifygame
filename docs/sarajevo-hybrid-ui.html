核心设计思路

原则：改5个CSS变量，不重写整套UI。
版本2的 --bg-* / --paper-* / --text-* 体系全部保留，只在顶层追加5个 --era-* 时代变量，每章切换时覆盖这5个变量即可。底色系统完全不动。

上方手机模拟器已实现完整的时代切换效果，点击七个章节按钮即可查看。

5个时代变量（接入点）

变量名 作用范围 影响的组件
--era-accent 时代主强调色 AP圆点、年份文字、事件卡竖线、按钮边框、底导选中色、进度条
--era-border 时代边框装饰色 顶栏底线、弹出面板左侧高亮线
--era-top-overlay 顶栏半透明叠加 顶栏 ::before 伪元素叠加色（传递年代氛围）
--era-texture 纹理CSS类名 通过JS切换 .texture-* class 控制背景纹理
--era-font-deco 装饰字体风格 用于年份、章节标题等显示字体

版本来源对照

  • V2保留 全局底色 --bg-base/--bg-surface/--paper-* 体系
  • V2保留 字体系统（Playfair Display + Noto Serif SC）
  • V2保留 严禁事项（禁止 Material Design / 禁止大圆角 / 禁止高饱和 emoji 图标）
  • V2保留 完整组件状态机（Default/Hover/Active/Disabled/Loading）
  • V2保留 动效禁令（禁止 bounce、禁止 spring 弹性）
  • V1接入 7套时代 accent 色，覆盖 --era-* 变量
  • V1接入 战争章节（2/4/6章）顶栏红色斜纹叠加
  • V1接入 第六章 UI 信号抖动干扰特效
  • V1接入 时代切换 0.6s 渐变过渡动画
  • V1接入 各章节对应的特色加载动画（实现时按章节注入）

工程实现建议

时代切换代码极简，只需在回合结算后执行：

document.documentElement.setAttribute('data-era', '3')

配合1.5秒叙事过渡卡片（全屏渐显），玩家完全感知时代更迭，但UI底层代码几乎不增加维护成本。
